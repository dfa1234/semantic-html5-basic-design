Why it matters

Advantages of sticking to basic design

  • Lightweight and loads fast
  • Fits on all screens
  • Looks the same in all browsers
  • Accessible to every browser, even the older ones
  • Legible and get one's point across (instead of diluting it with 5mb pics of hipsters drinking coffee :) )

Now adding some basic CSS basic rules to improve readability at zero costs can help improve readability a lot:

CSS basic rules

  • Because in the basic version lines are to close from each other and lack breathe: a minimum line-height 1.4 for body text, headings can be tighter.
  • Like it's the case for a physical book, the text should not touch the browser borders.
  • There is no true black on white in real life neither. Background #eee and foreground text #444 help to tone down.
  • Body font size should render close to 16px, or more. Less is only good for printed pages, not on screen.
  • Looking at screens require more efforts than on paper. Line should not never be longer than 200 characters long, and in average, 60 or 80 characters is good enough.

What is Semantic HTML

One of the most important features of HTML5 is its semantics. Semantic HTML refers to syntax that makes the HTML more comprehensible by better defining the different sections and layout of web pages. It makes web pages more informative and adaptable, allowing browsers and search engines to better interpret content .
